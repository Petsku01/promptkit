# GPT-4.1: Writing

## Text Improvement
```
Improve this text:

Goals:
- Fix grammar and spelling
- Improve clarity
- Make it more concise
- Maintain original tone

Provide:
1. Improved version
2. List of changes made

Text:
```

## Summary
```
Summarize this text:

Length: [X sentences / X words]
Focus: [key points / main argument / action items]
Audience: [technical / general / executive]

Text:
```

## Documentation
```
Write documentation for:

[paste code/API/feature]

Include:
- Overview (what it does)
- Usage examples with code snippets
- Parameters/options with types
- Return values
- Common errors and troubleshooting
- Changelog or migration notes if applicable
```

## Email Writing
```
Write an email:

Purpose: [request / follow-up / announcement / etc]
Tone: [formal / friendly / urgent]
Recipient: [colleague / client / manager]
Key points to include:
- [point 1]
- [point 2]

Keep it concise. No filler.
```

## Technical Blog Post
```
Write a technical blog post about [topic].

Audience: [junior developers / senior engineers / general tech]
Length: [short/medium/long]
Tone: [informative / opinionated / tutorial]

Structure:
1. Hook — why this matters
2. Problem or context
3. Solution or approach
4. Code examples
5. Key takeaways

Include working code examples.
```

---
**GPT-4.1 Writing Tips:**
- Follows formatting instructions precisely — specify tone, length, structure
- Better at maintaining consistent voice over long documents than GPT-4o
- Use explicit constraints ("no filler", "under 200 words") for best results
- Specify audience explicitly — it adapts language level well
