# Grok 3: Coding

## Quick Implementation
```
Write [language] code that does [task].

Keep it clean and straightforward. No over-engineering.
```

## Code Review (Direct Style)
```
Review this code. Be blunt about problems.

[paste code]

What sucks about it and how do I fix it?
```

## Debug This
```
This code is broken:

[paste code]

Error: [paste error]

What's wrong and how do I fix it? Skip the pleasantries.
```

## Architecture Discussion
```
I'm building [system description].

Current thinking:
[your approach]

Poke holes in this. What am I missing?
What would you do differently?
```

---
**Grok 3 Coding Tips:**
- Direct, no-nonsense style
- Won't sugar-coat feedback
- Good for quick iterations
- Handles real-world messy code well
