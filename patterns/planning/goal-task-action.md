# Goal-Task-Action Pattern

## Description

Structure planning by clearly separating goals, tasks, and specific actions.

## When to use

When delegating work to AI agents or breaking down complex projects into manageable pieces.

## When to avoid

Simple, well-understood tasks that don't need formal structure.

## Template

```
## Goal
[The high-level outcome we want to achieve]

## Tasks
To achieve this goal, we need to:
1. [Task 1]
2. [Task 2]
3. [Task 3]

## Current Focus
We are working on: [Current task]

## Actions Needed
For this task, take the following actions:
- [ ] [Action 1]
- [ ] [Action 2]
- [ ] [Action 3]

## Success Criteria
[How we'll know this is done correctly]

## Constraints
- [Constraint 1]
- [Constraint 2]
```

## Example

```
## Goal
Launch a new documentation website for our API that reduces support tickets by 30%.

## Tasks
1. Audit existing documentation gaps
2. Design information architecture
3. Write core API guides
4. Set up documentation platform
5. Add search functionality
6. Create getting started tutorial

## Current Focus
We are working on: Task 3 - Write core API guides

## Actions Needed
- [ ] Write authentication guide
- [ ] Write error handling guide
- [ ] Document rate limits
- [ ] Add code examples in Python, JavaScript, and Go

## Success Criteria
A developer new to our API can make their first successful call in under 10 minutes.

## Constraints
- Use existing brand guidelines
- Must work without JavaScript for accessibility
```
