# Bullet List Format Pattern

## Description

Request output as a structured bullet list for clarity and readability.

## When to use

When presenting multiple items, steps, or key points where order matters less than clarity.

## When to avoid

When hierarchical relationships are important (use numbered lists or tables instead).

## Template

```
[Task description]

Format your response as a bullet list where each item:
- Starts with a clear, concise statement
- Includes 1-2 sentences of explanation if needed
- Uses consistent formatting

Optional: Group related items under subheadings.
```

## Example

```
List the key benefits of using TypeScript over plain JavaScript.

Format your response as a bullet list where each item:
- Starts with a clear, concise statement
- Includes 1-2 sentences of explanation
- Uses consistent formatting

Group by category:
- Developer Experience
- Code Quality
- Maintainability
```
