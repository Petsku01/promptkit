# Negative Constraints Pattern

## Description

Explicitly specify what NOT to do or include, preventing common unwanted behaviors.

## When to use

When certain outputs are harmful, off-brand, or would waste the user's time.

## When to avoid

Over-constraining to the point of stifling useful creativity or nuance.

## Template

```
[Task description]

Important constraints - DO NOT:
- [Specific thing to avoid 1]
- [Specific thing to avoid 2]
- [Specific thing to avoid 3]
- [Specific thing to avoid 4]

If you cannot complete the task without violating these constraints, say so explicitly.
```

## Example

```
Write a social media post announcing our new feature.

Important constraints - DO NOT:
- Use exclamation marks (we're a professional B2B company)
- Include the word "revolutionary" or "game-changing" (avoid hyperbole)
- Mention competitors by name
- Make promises about future features we haven't committed to
- Use emojis

If you cannot complete the task without violating these constraints, say so explicitly.
```
