# Prompt-Analysis Pattern

## Description

Analyze an existing prompt to identify strengths, weaknesses, and improvement opportunities.

## When to use

When optimizing existing prompts, learning prompt engineering, or debugging why a prompt isn't working.

## When to avoid

New prompts where starting fresh is easier than analysis.

## Template

```
Analyze the following prompt and provide feedback:

```
[EXISTING_PROMPT]
```

For your analysis, cover:
1. **Clarity**: Is the request unambiguous?
2. **Context**: Is sufficient background provided?
3. **Constraints**: Are requirements specific enough?
4. **Output format**: Is the expected response defined?
5. **Edge cases**: What scenarios might cause issues?
6. **Missing elements**: What important details are absent?

Provide:
- **Rating**: Poor/Fair/Good/Excellent for each category
- **Issues**: Specific problems found
- **Recommendations**: Concrete improvements with examples
```
