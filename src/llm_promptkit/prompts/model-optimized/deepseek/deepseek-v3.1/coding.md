# DeepSeek V3.1: Coding

## Full-Stack Implementation
```
Implement this feature end-to-end:

Feature: [description]
Stack: [framework/language]
Existing codebase patterns: [describe or reference]

Steps:
1. Read relevant existing files for patterns
2. Design minimal changes
3. Implement backend (API, data model)
4. Implement frontend (UI, state management)
5. Add tests for each component
6. Verify everything works together

Follow existing code style. Keep changes focused.
```

## Code Review (Structured)
```
Review this code change:

[paste diff or code]

Evaluate:
1. **Correctness**: Does it do what's intended?
2. **Security**: Any injection, auth, or data exposure risks?
3. **Performance**: Unnecessary allocations or O() issues?
4. **Maintainability**: Readable, well-named, documented?
5. **Edge cases**: What inputs break it?

Rate each category: ✅ Good / ⚠️ Concern / 🔴 Blocker

Provide specific line-level suggestions.
```

## Debug & Fix
```
Debug this error:

Error: [paste error/stack trace]
Code: [paste relevant code]
Expected behavior: [what should happen]

Analysis:
1. What does the error mean literally?
2. Where in the code does it originate?
3. What's the root cause?
4. Minimal fix (don't over-engineer)
5. How to verify the fix works
```

## API Implementation
```
Design and implement a REST API for: [resource]

Technology: [Python/Node.js/etc.]
Database: [PostgreSQL/MongoDB/etc.]

Provide:
1. Endpoint specifications (method, path, params)
2. Request/response schemas with types
3. Implementation with error handling
4. Input validation
5. Authentication middleware
6. Tests for each endpoint

Use OpenAPI-compatible schemas.
```

## Code Refactoring
```
Refactor this code to improve: [readability/performance/maintainability]

[paste code]

Goals:

Constraints:
- Preserve existing behavior
- All current tests must pass
- No new dependencies unless necessary

Show before/after for each change with explanation.
```
