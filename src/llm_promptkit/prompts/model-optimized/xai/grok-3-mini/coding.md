# Grok 3 Mini: Coding

## Quick Code Generation
```
Write [language] code for: [task]

Keep it clean and straightforward. No over-engineering.
```

## Debug This
```
This code is broken:

[paste code]

Error: [paste error]

What's wrong and how do I fix it? Skip the pleasantries.
```

## Code Review (Fast)
```
Review this code. Focus on bugs and security issues.

[paste code]

What needs fixing? Skip style nits unless they indicate a real problem.
```

## Quick Script
```
Write a [language] script to: [task]

Requirements:
- [req 1]
- [req 2]

Don't over-engineer it. Just make it work.
```

## Explain Code
```
Explain what this code does in plain terms:

[paste code]

What does it do? What are the edge cases?
```

---

**Grok 3 Mini Coding Tips:**
- Great for quick code generation and simple debugging — fast and cheap
- Direct style: tell it what you want, it'll give it to you straight
- For complex multi-file work or deep architectural decisions, use full Grok 3 instead
- Thinking mode can help with harder bugs — adds reasoning depth at still-low cost
- Handles real-world messy code well — no need to sanitize inputs
- Perfect for CI/CD pipelines and automated code review at scale
