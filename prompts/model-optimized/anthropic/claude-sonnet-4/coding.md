# Claude Sonnet 4: Coding

## Code Generation
```xml
<task>
Write [language] code for [feature].
</task>

<requirements>
- [requirement 1]
- [requirement 2]
</requirements>

<constraints>
- [constraint 1]
- [constraint 2]
</constraints>

Include error handling and tests.
```

## Quick Code Review
```xml
<code>
[paste code]
</code>

Review for:
1. Bugs and logic errors
2. Security issues
3. Performance concerns
4. Code clarity

Be concise. Focus on actionable feedback.
```

## Debugging
```xml
<error>
[paste error message]
</error>

<code>
[paste relevant code]
</code>

<context>
[what you were trying to do]
</context>

Find the bug and provide a fix.
```

## API Implementation
```xml
<specification>
[API spec or requirements]
</specification>

<tech_stack>
- Language: [language]
- Framework: [framework]
- Database: [database]
</tech_stack>

Implement with:
- Input validation
- Error handling
- Proper HTTP status codes
- Basic documentation
```

---
**Claude Sonnet 4 Tips:**
- Best balance of speed and capability
- Good for most coding tasks
- XML structure improves output quality
- Fast enough for interactive use
- More cost-effective than Opus for routine tasks
